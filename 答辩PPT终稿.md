## P3 · 核心问题

### 三个绕不过去的坎

| 问题 | 表现 | 不做处理的后果 |
|------|------|-------------|
| **格式异构** | 成文法、判例、通知格式完全不同 | 统一切分打碎语义，检索失效 |
| **口语歧义** | "上海怎么管的" → 系统不知道"上海"指什么 | 搜出大量无关内容 |
| **零容忍幻觉** | 条文差一个字影响合规判断 | 不能像通用 ChatBot 自由发挥 |

### 本次聚焦四个模块

```
① 三轨分段      → 怎么让机器读懂不同格式的制度文件
② 元数据自动提取 → 如何从文件名挖出结构化标签
③ 实体检测      → 用户说"上海"，怎么自动锁定到具体机构和文件
④ 查询重写      → 口语问题怎么转成检索能听懂的查询
```

## P5 · 三轨分段

### 问题：固定字符切分破坏语义

```
固定 512 字符切分：
  Chunk A: "...公司章程另有规定。第一百四十八条 董事、监事..." ← 条文被切断
  Chunk B: "...应当遵守法律..." ← 不完整的条文后半段
```

### 方案：按文档类型选择切分策略

| doc_type | 策略 | 典型文档 |
|----------|------|---------|
| **law** | 强制按"第X条"切割，永不跨条 | 公司法、证券法 |
| **case** | 按裁判文书段落结构切分 | 判决书（当事人/本院认为/判决） |
| **other** | 三级自适配：第X条 → 一、二、三 → 递归 | 监管通知、指导意见 |

### other 三级自适配

```
文本进入
  → 第1级：正则"第[零一二三...\d]+条" → 命中 ≥2 段? → 按条输出
  → 第2级：匹配"一、""二、""（一）""（二）" → 命中 ≥2 段? → 按序号输出
  → 第3级：递归切分（\n\n → \n → 。→ ；→ ，）
核心原则：宁可切碎，不可跨结构边界
```

## P6 · 元数据自动提取

### 为什么需要

向量检索只知道文本像不像，不知道"上海银保监局发的"和"提到上海的"。需要结构化标签来精确过滤。

### 四个字段，全自动提取

```
文件名: "上海银保监局办公室关于印发《上海保险专业中介
         机构合规管理暂行办法》的通知_20230601.txt"

提取结果:
  law_name:      "上海保险专业中介机构合规管理暂行办法"  ← 《》内提取
  authority:     "上海银保监局"  ← "关于"前提取 → 去"办公室"后缀
  effective_date: "2023-06-01"  ← _YYYYMMDD → YYYY-MM-DD
  status:        "有效"  ← 多版本自动判定，最新=有效，其余=已修订
```

### 多版本自动管理

```
公司法_20131228 → "已修订"
公司法_20231229 → "有效"  ← 检索默认只看这个
```

**零人工标注，完全靠文件名解析规则。**

## P7 · 实体检测

### 问题

用户不会说"authority=上海银保监局"，只会说"上海怎么管的"。

### 方案：三层互补

```
用户问题: "上海的保险中介合规管理办法"
              │
  ┌───────────┼───────────┐
  ▼           ▼           ▼
第1层        第2层        第3层
金融词典     文件扫描索引  机构名索引
500+别名     自动构建      公文规则
```

| 层级 | 数据来源 | 示例 |
|------|---------|------|
| 词典 | 人工 + 自动 | "CAR"→"资本充足率" |
| 文件索引 | 启动时扫描全部 .txt | 《》内法规名 → 完整文件名 |
| 机构名索引 | "关于"前 = 发文机构 | "上海"→"上海银保监局,上海保监局" |

### 关键决策

```
优先级：文件名 > 法律名 > 机构名（从精确到模糊）
匹配算法：最长子串优先（避免"上海"误匹配为机构而非文件名中的"上海市..."）
```

## P8 · 查询重写

### 口语和检索语是两种语言

```
"我想了解一下保险公司在资本金方面有什么要求？"
              ↓ 重写
"保险公司资本金要求"
```

口语噪声："我想了解一下""在...方面""有什么"→ 稀释核心实体权重。

### 为什么不用大模型？

| | 0.5B 小模型 | 7B 大模型 |
|--|-----------|---------|
| 延迟 | <100ms | ~5s |
| 显存 | 极低 | 4.5GB |
| 任务适配 | ✅ 重写是表面变换 | ❌ 大材小用 |

### 方案：小模型 + 微调

```
Qwen2.5-0.5B（通用）→ LoRA 微调（600条金融改写数据）→ 领域重写器
Adapter 仅 ~1MB，运行时附加到基座模型
```

### 三级降级

```
小模型 0.5B（<100ms）→ 失败 → 主 LLM（~5s）→ 失败 → 不重写
        首选                     备选                兜底
```

## P9 · 四个模块的协同

```
用户: "上海银保监局怎么管保险中介的合规？"
  │
  ├── ③ 实体检测: "上海"→上海银保监局，"保险中介"→术语匹配
  │     → authority_filter, term_list
  │
  ├── ④ 查询重写: "上海保险专业中介机构合规管理办法"
  │     → rewritten_query
  │
  ├── 检索: authority="上海银保监局" + status="有效" + query=重写后
  │     → 精准命中目标 Chunk（② 元数据标记 + ① 切分完好）
  │
  └── LLM: 基于正确 Chunk 生成答案 + 来源溯源
```
