# 金融法规 RAG 智能问答系统 — 项目更新技术文档

## 一、项目简介

基于 RAG（Retrieval-Augmented Generation）的金融法规智能问答系统，支持 83 部中国金融法律文档的语义检索与智能问答。

**技术栈**：Streamlit（前端）+ FastAPI（后端）+ Milvus（向量数据库）+ MySQL（关系数据库）+ DashScope/DeepSeek（LLM API）

**核心能力**：向量语义检索 + BM25 关键词检索 + RRF 融合 + Reranker 精排 + LLM 流式生成 + 对话历史持久化 + 收藏功能

---

## 二、MySQL 历史查询与收藏

### 2.1 需求背景

系统原本无持久化存储，所有对话历史仅存在于 Streamlit session 中，页面刷新即丢失。需要实现：

- **对话历史持久化**：所有问答记录保存到 MySQL，支持跨会话查看
- **收藏功能**：支持收藏整个对话和单条溯源条文
- **实时流式问答 + 自动保存**：SSE 流式输出的同时自动持久化

### 2.2 数据库设计

三表结构，通过 SQLAlchemy ORM 管理：

```
┌─────────────────┐     ┌──────────────────────┐     ┌─────────────────┐
│  conversations  │     │      messages         │     │    favorites     │
├─────────────────┤     ├──────────────────────┤     ├─────────────────┤
│ id (UUID PK)    │◄────│ conversation_id (FK) │     │ id (UUID PK)    │
│ title           │     │ role (user/assistant)│     │ fav_type (enum) │
│ created_at      │     │ content              │     │ conversation_id │
│ updated_at      │     │ question             │     │ message_id      │
└─────────────────┘     │ rewritten_query      │     │ source_data     │
                        │ sources (JSON)       │     │ note            │
                        │ confidence (JSON)    │     │ created_at      │
                        │ created_at           │     └─────────────────┘
                        └──────────────────────┘
```

**关键设计**：
- `conversations` 级联删除：删除对话时自动删除关联的 `messages` 和 `favorites`
- `sources` / `confidence` 使用 MySQL JSON 列存储，便于前端结构化渲染
- `favorites.fav_type` 枚举：`conversation`（对话收藏）/ `source`（条文收藏）

### 2.3 后端 API 接口

| 方法 | 路径 | 功能 |
|------|------|------|
| `POST` | `/api/conversations` | 创建对话 |
| `GET` | `/api/conversations` | 获取对话列表（按 `updated_at` 倒序） |
| `GET` | `/api/conversations/{id}` | 获取对话详情（含全部消息） |
| `DELETE` | `/api/conversations/{id}` | 删除对话（级联删除消息和收藏） |
| `POST` | `/api/favorites` | 添加收藏 |
| `GET` | `/api/favorites` | 获取收藏列表（支持 `fav_type` 过滤） |
| `DELETE` | `/api/favorites/{id}` | 取消收藏 |
| `POST` | `/api/qa/stream` | SSE 流式问答（自动保存对话历史） |

### 2.4 流式问答自动保存

`/api/qa/stream` 端点在流式输出的同时自动完成持久化：

```
1. 创建/获取 conversation
2. 保存用户消息（user role）
3. 流式输出 RAG 回答（逐 token 推送 SSE 事件）
4. 流式结束后，保存助手消息（assistant role）含 sources + confidence
5. 返回 [DONE] 事件
```

### 2.5 前端交互

**侧边栏功能**：
- **对话历史列表**：显示最近对话，支持切换、删除
- **收藏面板**：显示最近 10 条收藏，支持取消收藏
- **收藏按钮**：对话区域和溯源条文旁均设有收藏按钮

### 2.6 解决的关键问题

#### 问题 1：MySQL JSON 列序列化

pymysql 返回 MySQL JSON 列时类型为 `str` 而非 `dict`，导致 Pydantic 模型校验失败。

**解决方案**：在 API 层统一增加类型检查和 `json.loads()` 转换：

```python
sd = fav.source_data
if isinstance(sd, str):
    try:
        sd = json.loads(sd)
    except (json.JSONDecodeError, TypeError):
        pass
```

#### 问题 2：init_db() 建表失败

原始代码在 `api_app.py` 中调用 `init_db()` 时未导入 ORM 模型，导致 SQLAlchemy 无法识别表结构。

**解决方案**：在 `init_db()` 调用前先导入 `models` 模块：

```bash
# docker-entrypoint.sh
python -c "import src.models; from src.database import init_db; init_db()"
```

#### 问题 3：收藏列表重复拉取

每次 Streamlit 页面重跑都会调用 `fetch_favorites()`，造成不必要的 HTTP 请求。

**解决方案**：引入 `session_state.favorites_list` 缓存，仅在首次加载和增删操作后刷新：

```python
if st.session_state.favorites_list is None:
    st.session_state.favorites_list = fetch_favorites(api_base)

# 收藏/取消收藏后刷新缓存
st.session_state.favorites_list = None  # 下次重跑重新拉取
```

### 3.2 镜像构建

**基础镜像**：`pytorch/pytorch:2.11.0-cuda12.6-cudnn9-runtime`（支持 GPU 推理）

**构建时下载模型**：
- `BAAI/bge-small-zh-v1.5`（Embedding）
- `BAAI/bge-reranker-v2-m3`（Reranker）
- `Qwen/Qwen2.5-7B-Instruct-GPTQ-Int4`（本地 LLM，约 5.6GB）

**国内加速**：
- 所有 Docker 镜像使用 DaoCloud 镜像加速（`docker.m.daocloud.io`）
- Python 依赖使用清华 PyPI 镜像
- HuggingFace 设置 `HF_ENDPOINT=https://hf-mirror.com`

### 3.3 服务配置

| 服务 | 镜像 | 端口 | 依赖 |
|------|------|------|------|
| etcd | `quay.io/coreos/etcd:v3.5.18` | 2379 | 无 |
| MinIO | `minio/minio:RELEASE.2024-12-18` | 9001 | 无 |
| Milvus | `milvusdb/milvus:v2.5.10` | 19530 | etcd, MinIO |
| MySQL | `mysql:8.0` | 3306 | 无 |
| API | 自建 `rag-finance-api` | 8000 | MySQL, Milvus |
| Frontend | 自建 `rag-finance-api` | 8501 | API |

### 3.4 环境变量管理

支持两套配置文件：
- `.env`：本地开发环境
- `.env.docker`：Docker 容器内部

**LLM 三级降级策略**：

```
1. 本地 Qwen2.5-7B-Int4（需 GPU + 模型文件）
   ↓ 无 GPU 或模型缺失
2. DeepSeek API（deepseek-chat）
   ↓ API Key 未设置
3. DashScope API（qwen-plus）
   ↓ API Key 未设置
4. RuntimeError: 无可用 LLM
```

### 3.5 启动流程

`docker-entrypoint.sh` 自动化启动：

```bash
# 1. 初始化 MySQL 表结构
python -c "import src.models; from src.database import init_db; init_db()"

# 2. 启动 FastAPI 后端（后台进程）
uvicorn rag_finance_system.api_app:app --host 0.0.0.0 --port 8000 &

# 3. 等待 API 就绪（最多 60 秒）
for i in $(seq 1 30); do
    if curl -sf http://localhost:8000/openapi.json > /dev/null 2>&1; then
        break
    fi
    sleep 2
done

# 4. 启动 Streamlit 前端
streamlit run rag_finance_system/app.py --server.port 8501
```

### 3.6 解决的关键问题

#### 问题 1：Docker 镜像体积过大

Qwen2.5-7B-Int4 模型文件约 5.6GB，加上 PyTorch 基础镜像，总镜像超过 10GB，构建时间过长且网络不稳定。

**解决方案**：切换为本地开发模式，基础设施（MySQL、Milvus、etcd、MinIO）使用 Docker，应用层（API + Frontend）在本地 Python 环境运行。

#### 问题 2：HuggingFace 下载阻塞

国内环境无法直接访问 HuggingFace 下载模型。

**解决方案**：设置 HuggingFace 镜像环境变量：

```bash
HF_ENDPOINT=https://hf-mirror.com
```

#### 问题 3：本地 LLM 无 GPU 降级

本地环境无 GPU，加载 Qwen2.5-7B-Int4 会卡死。

**解决方案**：`get_llm()` 函数增加 GPU 检查，无 GPU 时跳过本地模型，自动降级到 API：

```python
def get_llm(prefer_local=True):
    if prefer_local and Path(LLM_MODEL_PATH).exists():
        try:
            import torch
            if torch.cuda.is_available():
                return LocalLLM(...)
        except Exception:
            pass
    # 降级到 API
    return APILLM(...)
```

## 四、网页刷新机制优化

### 4.1 问题分析

Streamlit 的交互模型是"每次操作触发完整页面重跑"，导致：

1. **不必要的 HTTP 请求**：每次页面重跑都调用 `fetch_favorites()` 和 `fetch_conversations()`
2. **`st.rerun()` 滥用**：切换对话、删除操作后调用 `st.rerun()` 导致二次重跑和闪烁
3. **健康检查频繁**：每次重跑都调用 `/health` 接口检查 API 和 MySQL 状态

### 4.2 优化方案

#### 4.2.1 健康检查缓存

引入 30 秒缓存窗口，避免每次重跑都调用健康检查接口：

```python
_health_cache = {"ts": 0.0, "api_ok": False, "mysql_ok": False}

def cached_health_check(api_base):
    now = time.time()
    if now - _health_cache["ts"] < 30:
        return _health_cache["api_ok"], _health_cache["mysql_ok"]
    # 实际检查...
    _health_cache.update(ts=now, api_ok=api_ok, mysql_ok=mysql_ok)
    return api_ok, mysql_ok
```

#### 4.2.2 收藏列表缓存

收藏列表仅在以下情况刷新：
- 首次加载（`favorites_list is None`）
- 用户点击刷新按钮
- 用户收藏或取消收藏条文

```python
# 首次加载
if st.session_state.favorites_list is None:
    st.session_state.favorites_list = fetch_favorites(api_base)

# 收藏/取消收藏后失效缓存
st.session_state.favorites_list = None
```

#### 4.2.3 移除不必要的 `st.rerun()`

| 操作 | 优化前 | 优化后 |
|------|--------|--------|
| 新对话 | `st.rerun()` | 仅状态更新 |
| 切换对话 | `st.rerun()` | 仅状态更新 |
| 删除对话 | `st.rerun()` | 仅状态更新 |
| 刷新列表 | `st.rerun()` | 保留（用户主动触发） |
| 取消收藏 | `st.rerun()` | 设缓存失效，自动重跑 |

Streamlit 的 widget 回调本身会触发重跑，额外调用 `st.rerun()` 会导致二次重跑和界面闪烁。

### 4.3 实现效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 每次交互 HTTP 请求 | 3-5 次 | 1-2 次 |
| 健康检查频率 | 每次重跑 | 30 秒缓存 |
| 收藏列表拉取 | 每次重跑 | 仅增删后刷新 |
| 页面闪烁 | 频繁（st.rerun） | 消除 |
